งานที่ได้รับฝึกวันนี้ คือการสร้าง เว็บที่จำลองการทำงานร่วมกันระหว่าง Frontend + Backend + Localstorage
เริ่มแรก ผมทำการมองว่า เว็บนี้มีกี้หน้าก่อน รวมๆ แล้วใช้ 2หน้า คือ home/user page & ownerPage 
ลำดับต่อมา @homepage ผมเลือกแบ่งองค์ประกอบออกเป็นส่วนๆ เพราะ เดิมการเขียนอยู่ในหน้าเดียวกันนั้น มันง่ายและสบายกว่า แต่การ maintain นั้นทำได้ยาก เนื่องจากมันเป็นภาพใหญ่ แต่ปัญหาที่เกิดมันคือจุดเล็กๆ ที่กระทบกันไปเป็นระบบ ดังนั้น การแบ่งออกเป็น component จะช่วยแก้ปัญหาเรื่องนี้ได้ และ ลดผลกระทบดังกล่าว" 
ผมเริ่มแบ่งหน้า homepage เป็นลำดับดังนี้
   1.   Navbar มีปุ่มที่ต้องกดไปหน้าไปอีกสองปุ่ม คือ home[หน้าแรก] owner[หน้าของผู้ดูแลระบบหลังบ้าน]
   2.   state of button [เพื่อเช็คว่าเป็นใคร permission priorty]
   3.   OwnerPage 
   4.   UserButton
   5.   OwnerButton

ตัวแปร state = sector : ดึงข้อมูลมาโชว์ตอนเราเลือกปุ่ม ที่แสดงว่าเลือก role ไหน
,member : ใช้เก็บข้อมูลเมื่อเรากรอกชื่อและกดบันทึก [role admin],
fromdata : ใช้ดึงข้อมูลมาจาก API เช่น firstname lastname postion 
ีAI ช่วยผมใช้ useffect มาจัดการ ในด้านของAdminและโชว์ข้อมูลที่ดึงมาจาก API ในส่วนของ user 
ส่วน Admin จะใช้ในการ จัดการตาราง เพิ่มฟังชั่น สร้างรายชื่อ/พนักงานใหม่และสามารถลบรายชื่อในตารางได้ กำหนดโรล
fetch ไม่ใช้กับ useeffect คิดว่าได้ครับ แต่อาจจะทำให้รันงานได้ช้าลง เพราะไม่มีคนนจัดการ อันนี้หาข้อมูลเพิ่มมาครับ
fetch ควรใช้ asyn เพราะทุกprocess จะรันได้พร้อมกันในระหว่างที่ ดึง API และ เปลี่ยนแปลงข้อมูล เนื่องจากเมื่อมีการดึงข้อมูลจากAPI และเราทำการเปลี่ยนแปลงข้อมูล มันเป็นกระบวนการที่เกิดขึ้นพร้อมกัน 
สรุป ความเข้าใจในเรื่อง react จากการที่ทำวันนี้พบ ว่าพอนึกว่าต้องทำตาราง ทำปุ่มแยก แยกส่วน และลิงค์ แต่ยังเขียนออกมาไม่ได้ และมี gap เยอะอยู่
ในส่วนการทำงานรู้สึกว่าพอเห็นการทำงานร่วมกันระหว่าง หน้าบ้านและหลังบ้าน รวมถึงการเก็บข้อมูลใน localStorage
